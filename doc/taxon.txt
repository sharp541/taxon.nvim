*taxon.txt*  taxon.nvim

taxon.nvim is a note management plugin for Neovim built around hierarchical
tags inspired by taxonomy.

==============================================================================
OVERVIEW                                                    *taxon-overview*

Taxon stores one Markdown note per file in |taxon-notes_dir|. Query commands
rescan the directory on each invocation, derive parent tags in memory, and do
not write a cache or index file in the MVP.

|:TaxonTitleSearch| and |:TaxonTagSearch| require
`nvim-telescope/telescope.nvim`. The rest of the MVP command surface does not.

==============================================================================
COMMANDS                                                  *taxon-commands*

:TaxonOpen                                                      *:TaxonOpen*
    Create |taxon-notes_dir| if needed and open the configured notes
    directory.

:TaxonNew                                                        *:TaxonNew*
    Prompt for a title, create a new note in |taxon-notes_dir|, and open it.
    Titles are preserved in the timestamped filename unless they contain
    filename-unsafe characters such as `/` or `\`.

:TaxonTitleSearch                                        *:TaxonTitleSearch*
    Rescan |taxon-notes_dir|, search note titles with Telescope, and open the
    selected note.
    If Telescope is unavailable, Taxon shows a clear error message.

:TaxonTagSearch                                            *:TaxonTagSearch*
    Rescan |taxon-notes_dir|, search normalized explicit and inherited tags
    with Telescope, then choose and open a matching note.
    If Telescope is unavailable, Taxon shows a clear error message.

:TaxonTagTree                                                *:TaxonTagTree*
    Open the hierarchical tag tree in a dedicated vertical buffer.
    Press <CR> on a tag to open one of its matching notes, or `q` to close
    the tree view.

==============================================================================
LUA API                                                    *taxon-lua-api*

require('taxon').setup({ opts })                           *taxon-setup*
    Store plugin configuration and ensure |taxon-notes_dir| exists.

    If omitted, `notes_dir` defaults to:
>
        vim.fn.stdpath('data') .. '/taxon-notes'
<

require('taxon').open()                                     *taxon-open*
    Create |taxon-notes_dir| if needed and open the configured notes
    directory.

require('taxon').create_note(title)                   *taxon-create_note*
    Create a new note directly and return its full path.

    New notes use the canonical Taxon note template and filenames in the form
    `YYYYMMDD-HHMMSS-タイトル.md`.

require('taxon').new_note()                             *taxon-new_note*
    Prompt for a title and create a new note through the same flow used by
    |:TaxonNew|.

require('taxon').scan_notes()                           *taxon-scan_notes*
    Rescan |taxon-notes_dir| and return a query model with `notes`, `tags`,
    `tag_tree`, `notes_by_title`, `notes_by_tag`, and `invalid_notes`.

    Each note entry includes `path`, `title`, `explicit_tags`, and `tags`.
    The `tags` field contains both explicit frontmatter tags and inherited
    parent tags. Invalid Markdown notes are skipped deterministically and
    reported through `invalid_notes`.

    `tag_tree` is a deterministic list of root tag nodes. Each node includes
    `name`, `tag`, `notes`, and `children`.

require('taxon').search_titles()                     *taxon-search_titles*
    Rescan |taxon-notes_dir|, feed note titles into a Telescope picker, and
    open the selected note.

    If Telescope is unavailable, Taxon shows a clear error message instead of
    raising a raw Lua stack trace.

require('taxon').search_tags()                         *taxon-search_tags*
    Rescan |taxon-notes_dir|, feed normalized explicit and inherited tags into
    a Telescope picker, then show the notes that match the selected tag and
    open the selected note.

    If Telescope is unavailable, Taxon shows a clear error message instead of
    raising a raw Lua stack trace.

require('taxon').show_tag_tree()                     *taxon-show_tag_tree*
    Rescan |taxon-notes_dir| and render the deterministic tag hierarchy in a
    dedicated vertical buffer.

    Press <CR> on a tag to open its only matching note directly, or choose
    from the matching notes when the tag contains more than one note. Press
    `q` to close the tree buffer.

==============================================================================
NOTE FORMAT                                            *taxon-note-format*

Notes use YAML frontmatter with a `tags` list and the first Markdown H1 as the
title.

New notes are created with this shape:

>
    ---
    tags: []
    ---

    # Title
<

New notes use timestamp-prefixed filenames in the form
`YYYYMMDD-HHMMSS-タイトル.md`. Titles are preserved in the filename, but
filename-unsafe characters such as `/` and `\` are rejected.

Tags are normalized to lowercase slash-delimited paths when Taxon reads them.
Whitespace around each segment is trimmed. Spaces inside a segment and Japanese
text are allowed. Empty segments, leading or trailing `/`, `//`, control
characters, and newlines are rejected.

==============================================================================
CONFIGURATION                                        *taxon-configuration*

>
    require('taxon').setup({
      notes_dir = vim.fn.stdpath('data') .. '/taxon-notes',
    })
<

`notes_dir`                                              *taxon-notes_dir*
    Directory used to store Taxon-managed notes.
